OAuth 2.0 Auth Flow

  1. 1 Redirect Merchant Send user to TikTok Auth URL with App Key and Redirect URI.
  2. 2 Capture Auth Code TikTok redirects back to your server with a temporary code parameter.
  3. 3 Request Access Token POST code, app_key, and app_secret to exchange for access/refresh tokens.

Webhook Event Map

ORDER_STATUS_CHANGE

Triggers on payment confirmation, cancellations, and fulfillment updates.

PRODUCT_AUDIT_RESULT

Fires when products pass or fail regulatory or content moderation reviews.

REVERSE_ORDER_NOTIFY

Informs your integration when a customer requests a return or refund.

Critical Guardrails

  • Rate Limits: Global defaults sit at 10 to 20 Requests Per Second (RPS) per developer app.
  • Signatures: Every request requires an HMAC-SHA256 signature built using the URL path and your App Secret.
  • Token Longevity: Access tokens expire after 24 hours. Refresh tokens remain valid for up to one year.
